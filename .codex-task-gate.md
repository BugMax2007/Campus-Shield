# Codex Task Gate

This file is the project-local gate for development behavior. It intentionally replaces the idea of using `.DS_Store`, because `.DS_Store` is a macOS cache file and should not be used as a human-readable workflow document.

## Core Rule

Do not call a module "done" until both the logic path and the visual path are checked.

## Required Checks Before Closing Any Module

1. Scope check
- State the exact module or subsystem being changed.
- Confirm whether the change touches gameplay, data, UI, save data, or localization.

2. UI visual gate
- Launch the current build and inspect the affected screen or flow visually.
- If the module touches UI, check the result at `1280x720`, `1600x900`, and `1920x1080`.
- Confirm the anchor layout still holds: objective top-left, alert top-center, minimap top-right, prompt bottom-center, subtitles bottom-left, modal centered.
- Confirm Chinese text is not truncated and English support text remains readable.
- Confirm the affected interaction is still visible, discoverable, and keyboard-usable.

3. Logic gate
- Confirm the data contract still matches the code path.
- If a rule changed, verify the rule is not hardcoded in the render layer.
- If a score path changed, verify the category cap still holds.
- If a safe-room rule changed, verify both positive and negative cases.

4. Content gate
- Confirm the new content teaches a concrete safety concept.
- Confirm the content stays non-graphic, non-tactical, and educational.
- Confirm the content does not reward risky curiosity after the alert begins.

5. Regression gate
- Run automated tests relevant to the changed module.
- Manually verify one normal path and one failure or edge path.
- If the module changes save data or localization, verify load/save and missing-key behavior.

## Definition Of Done

A module is only complete when:

- the code path works,
- the relevant UI has been visually checked,
- the interaction is understandable without guessing,
- the localization still renders correctly,
- the test or manual verification note is updated in the current task summary.
